# HR Analytics Pipeline

End-to-end HR data pipeline and analysis built as a portfolio project,
demonstrating the full data analyst workflow: synthetic data generation,
realistic data quality injection, ETL, exploratory analysis, and people
analytics with statistical validation.

> 🚧 **Work in progress** — data generation and ETL scripts are complete.
> Analysis notebook and Power BI dashboard in progress.

**Dataset:** 5,000 employees · 1,500 gigs · 2023–2025 · 30 countries  
**Author:** Carlos Hisamitsu · [linkedin.com/in/carloshisamitsu](https://linkedin.com/in/carloshisamitsu)

## Reproducing the data

All data folders are gitignored — nothing in `data/` is committed.
Run these two scripts to regenerate everything locally.

### Prerequisites

Install [uv](https://docs.astral.sh/uv/):

```bash
# macOS / Linux
curl -LsSf https://astral.sh/uv/install.sh | sh

# Windows (PowerShell)
powershell -ExecutionPolicy ByPass -c "irm https://astral.sh/uv/install.ps1 | iex"
```

### Clone and install

```bash
git clone https://github.com/chisamitsu/hr-analytics-pipeline.git
cd hr-analytics-pipeline
```

**Option A — uv (recommended):**

```bash
uv sync
```

**Option B — pip:**

```bash
python -m venv .venv

# Windows
.venv\Scripts\activate

# macOS / Linux
source .venv/bin/activate

pip install -r requirements.txt
```

Then generate and run as normal. All `uv run python` commands in this README
can be replaced with plain `python` if you activated the venv via pip.

### Generate

```bash
# Step 1 — clean synthetic dataset → data/clean/ (11 CSV files)
uv run python scripts/generate_data.py

# Step 2 — inject data quality issues → data/dirty/ (11 CSV files + injection_manifest.csv)
uv run python scripts/inject_dirty_data.py --input data/clean --output data/dirty --seed 42
```

## Dataset overview

Fully synthetic dataset generated by `scripts/generate_data.py`.
Modelled on a real vendor SFTP → cloud data warehouse → Power BI pipeline.

| File | Rows | Description |
|---|---|---|
| `raw_employee_master.csv` | 5,000 | One row per employee |
| `raw_employee_job_assignment.csv` | ~8,700 | SCD2 job history |
| `dim_skill.csv` | 80 | Skill catalogue |
| `raw_employee_skills.csv` | ~92,000 | Employee-skill records |
| `raw_gig_master.csv` | 1,500 | Gig catalogue |
| `raw_gig_required_skills.csv` | ~6,000 | Gig-skill requirements |
| `raw_gig_applications_and_assignments.csv` | ~23,600 | Applications |
| `raw_user_activity_log.csv` | ~549,000 | Platform events |
| `raw_training_master.csv` | 80 | Training catalogue |
| `raw_training_skills.csv` | ~220 | Training-skill mappings |
| `raw_training_records.csv` | ~41,700 | Completion records |

The `data/dirty/` layer adds 34 realistic data quality issues across 8 categories
(encoding errors, duplicate rows, temporal inconsistencies, referential integrity
breaks, out-of-range values, business rule violations, and system noise) documented
in `data/dirty/injection_manifest.csv`.
